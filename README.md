## рҹ’Ў Why OPTIM?

Most image optimization tools require you to upload your files to a third-party server вҖ” exposing your images to unknown infrastructure, raising privacy concerns, and introducing latency. **OPTIM eliminates all of that.**

Everything happens directly inside your browser using the native Canvas API. Your images never leave your device. Period.

## вңЁ Features

### рҹ–јпёҸ Input & Preview
- **Drag & drop** or click-to-browse file selection
- Supported formats: **JPG В· PNG В· WebP В· BMP В· TIFF**
- Instant preview on load вҖ” no processing delay
- Three preview modes:
  - `Original` вҖ” raw input image
  - `Optimized` вҖ” processed output
  - `Compare вҶ”` вҖ” interactive drag slider for before/after

### рҹ“Ұ Output Formats
| Format | Best for |
|---|---|
| **JPG** | Photos, complex scenes, wide compatibility |
| **WebP** | Modern web, best size-to-quality ratio |
| **PNG** | Transparency, lossless graphics, screenshots |

> Automatic format suggestion based on the input file type.

### вҡҷпёҸ Compression Engine

Quality slider from **1 вҶ’ 100** with contextual labels:

```
1 в”Җв”Җв”Җв”Җ 25 в”Җв”Җв”Җв”Җ 50 в”Җв”Җв”Җв”Җ 75 в”Җв”Җв”Җв”Җ 100
Low   Medium  Standard  High   Max
```

Four compression methods:

| Method | Description |
|---|---|
| **Auto** | Recommended balanced mode вҖ” best for most use cases |
| **Aggressive** | Maximum file size reduction вҖ” may reduce quality |
| **Lossless** | Zero quality loss вҖ” larger output file |
| **Balanced** | Slight quality trade-off for significantly better size |

### рҹ“җ Resize & Dimension Control
- Enable/disable resize independently from compression
- Manual **width Г— height** inputs with live feedback
- **Aspect ratio lock** вҖ” maintains proportions automatically
- Resize algorithm selector:
  - `Bicubic` вҖ” smooth, professional-grade scaling
  - `High Quality Smooth` вҖ” anti-aliased output
  - `Pixelated` вҖ” crisp edges for pixel art / icons

Quick presets for common targets:

```
[ 1080p ] [ 720p ] [ 800px ] [ 400px ] [ Thumbnail ] [ Г—ВҪ ]
```

### рҹҺЁ Image Adjustments
All adjustments are applied via direct pixel manipulation вҖ” no filters, no approximations:

| Control | Method |
|---|---|
| **Sharpness** | Convolution kernel sharpening |
| **Brightness** | Per-channel pixel-level offset |
| **Contrast** | Histogram stretching |
| **Saturation** | Per-channel desaturation blend |

### рҹ”§ Advanced Options

| Option | Effect |
|---|---|
| **Strip EXIF metadata** | Removes GPS coordinates, device model, timestamps |
| **Progressive JPEG** | Enables progressive loading for web delivery |
| **White background fill** | Replaces transparency when exporting PNG вҶ’ JPG |

### рҹ“Ҡ Live Stats & Histogram
- Side-by-side file size comparison (original vs. optimized)
- **Savings percentage** displayed in real time
- Output dimensions confirmation
- **RGB histogram** updated live as you adjust settings

## рҹ”’ Privacy

OPTIM was designed with a **privacy-first** philosophy from the ground up.

- рҹ”җ **No upload** вҖ” images are never sent to any server
- рҹ“Ў **No analytics** вҖ” no tracking pixels, no session recording, no telemetry
- рҹҚӘ **No cookies** вҖ” nothing is stored outside your local session
- рҹҢҗ **Works fully offline** вҖ” after the initial page load, no network required
- рҹ§№ **EXIF stripping** вҖ” optionally removes all embedded metadata before export

All processing is handled by the [Canvas API](https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API) вҖ” a sandboxed, read-only browser API. Your images are processed in memory and immediately discarded when you close the tab.
